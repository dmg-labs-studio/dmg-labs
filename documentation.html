00 — Overview

DMG Labs Design System

This documentation covers every visual building block of the DMG Labs website. The system follows Atomic Design — small, reusable pieces compose into larger patterns. Edit one CSS variable in styles.css and the change propagates everywhere.

// FILE STRUCTURE

The site consists of four HTML pages (index.html, devices.html, guides.html, studio.html), a shared stylesheet styles.css, and this documentation file. Each HTML page already has its CSS fully inlined for portability — but new pages you create should link styles.css externally.

Atomic design hierarchy

Organisms — Nav, Hero, Green Band, CTA Section, Footer
Molecules — Product Card, Guide Card, Retro Window, Spec Table
Atoms — Buttons, Badges, Tags, Inputs, Decorators
Tokens — Colors, Fonts, Spacing, Shadows in :root

Linking styles.css on a new page

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>My New Page — DMG Labs</title>

  <!-- 1. Google Fonts (required) -->
  <link rel="preconnect" href="https://fonts.googleapis.com"/>
  <link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:ital,wght@0,300;0,600;0,700;1,300;1,600&family=DM+Mono:wght@300;400&family=Nunito:wght@400;600;700&family=Press+Start+2P&display=swap" rel="stylesheet"/>

  <!-- 2. DMG Labs design system -->
  <link rel="stylesheet" href="styles.css"/>
</head>

// HOW TO CHANGE THE ACCENT COLOR

Open styles.css. In the :root block at the top, update these three lines:

--color-orange: #f06a00; → your new color
--color-orange-lt: #ff8c2a; → ~20% lighter
--color-orange-dk: #c45500; → ~20% darker (used for shadows)

Color usage rules

✓ DO

Use orange only as an accent — on interactive elements (buttons, links, hover states), pixel labels, and the ticker strip. It should create contrast, not fill large areas.

▶ CTA Button PROTOTYPE Clock
✗ DON'T

Don't use orange as a section background color, or for body text. Don't mix multiple accent colors — pick orange or green for each element, not both on the same component.

Background sections should use --color-forest or --color-cream, not --color-orange.

02 — Foundation / Typography

Type System

Three typefaces, each with a distinct role. Never mix roles — don't use Press Start 2P for body text, don't use Nunito for pixel labels.

Cormorant Garamond

Italic, 300 weight

h1 – h4, taglines, pull quotes, hero titles

BODY / SANS

Nunito Bold

Body text, buttons, descriptions, all paragraph content

How to add a new photo to a device

// STEPS

1. Save your photo as a .jpg or .png in the img/ folder next to devices.html.
2. Open devices.html. Find the carousel for your device — look for id="carousel-ph1", id="carousel-d40", id="carousel-lu1", or id="carousel-nautilus".
3. Inside .img-carousel__track, replace a placeholder slide with a real slide:

Replace: <div class="img-carousel__slide img-carousel__slide--placeholder">...</div>
With: 

<div class="img-carousel__slide">
  <img src="img/your-new-photo.jpg"
       alt="Descriptive alt text"
       style="width:100%;height:100%;object-fit:cover;display:block;
              object-position:center"/>
  <div class="img-carousel__scanline"></div>
</div>

4. Update the dot count in .img-carousel__dots — add or remove <div class="img-carousel__dot"></div> elements to match the number of slides.
5. Update the counter text: <div class="img-carousel__counter">1 / 3</div> — change the 3 to your new slide count.
6. Save and reload. The JavaScript initialises automatically.

CSS classes at a glance

ClassRole
.img-carouselOuter wrapper. Set background to match the device’s dark color.
.img-carousel__trackFlex row that slides horizontally. Do not add padding or gap.
.img-carousel__slideOne slide — always flex:0 0 100%. Contains an <img> and the scanline overlay.
.img-carousel__slide--placeholderModifier. Shows a dashed crosshatch until a real image is added.
.img-carousel__scanlineCRT scanline overlay — always the last child inside a real slide.
.img-carousel__btn--prev / --nextArrow buttons. Remove both elements if the carousel has only one slide.
.img-carousel__counter“1 / 3” badge top-right. Remove if single slide.
.img-carousel__dotsContainer for dot indicators. One .img-carousel__dot per slide.
.img-carousel__dot.activeAdded by JS to the current dot. Do not set manually — JS handles it.

Interaction summary

InteractionBehaviour
Click ◀ / ▶ arrowsGoes to previous / next slide. Wraps around.
Click a dotJumps directly to that slide.
Touch swipe (mobile)Swipe left → next slide. Swipe right → previous.
Keyboard ← / →Click the carousel to focus it, then use arrow keys.

Video Player

The video slide is a drop-in addition to any .img-carousel. Add it as the last slide in the track. It shares the same dimensions, scanline overlay, and nav as image slides, and layers a play/pause overlay plus a mute toggle on top of a native <video> element. Clicking the overlay or the video itself toggles playback.

<div class="img-carousel__dots">

  <div class="img-carousel__dot"></div>  <!-- new dot for video slide -->

<div class="img-carousel__counter">1 / 4</div>  <!-- update total -->
// STEPS — adding a video to a device

1. Place your video file in a video/ folder next to devices.html — e.g. video/dmg40_demo.mp4. H.264 MP4 is the safest format for broad browser support.

2. Find the correct carousel in devices.html — look for id="carousel-ph1", id="carousel-d40", id="carousel-lu1", or id="carousel-nautilus".

3. Append a .img-carousel__slide--video block inside .img-carousel__track (shown above).

4. Add one <div class="img-carousel__dot"></div> to .img-carousel__dots.

5. Update the counter text to reflect the new total. The carousel JS handles everything else automatically.

CSS classes at a glance

ClassRole
.img-carousel__slide--videoModifier on .img-carousel__slide. Sets background:#000 and positions child elements.
.carousel-video-playFull-slide overlay. Visible when paused, hidden (.hidden) when playing. Toggled by JS on play/pause events.
.carousel-video-play__iconCircular orange play button inside the overlay.
.carousel-video-play__label“PLAY VIDEO” label in Press Start 2P below the icon.
.carousel-video-muteMute/unmute button anchored bottom-right. Hidden by default; .visible is added by JS while the video plays.

Playback behaviour

InteractionBehaviour
Click play overlayStarts playback. Overlay fades out; mute button appears.
Click video (while playing)Pauses. Overlay reappears; mute button hides.
Click video (while paused)Resumes playback.
Navigate away (arrow / dot / swipe)Video pauses automatically.
Navigate backIf video was started at least once (currentTime > 0), it auto-resumes. Otherwise the play overlay is shown.
Click mute / unmuteToggles video.muted. Label updates to 🔋 UNMUTE and back.
✓ One video slide per device

Keep the video as the final slide so users see images first. Use a poster= thumbnail that matches the device’s first photo for visual consistency.

✗ Multiple video slides in one carousel

The carousel pauses the currently visible video on navigation. Two video slides in the same carousel would require additional state tracking and risks unexpected playback overlap.

10f — Molecules / Guide Callouts

Guide Callouts

Left-border callout boxes for contextual notes inside guide steps. Three semantic variants: default orange (general notes), tip (green, positive guidance), warn (red, risks or requirements).

// NOTE
Use this for general clarifications that don’t fit in the main prose — alternative methods, optional steps, or context that’s good to know but not essential.
// TIP
Positive guidance, shortcuts, or recommended practices. Signals a better path the reader might not have found on their own.
// WARNING
Risks, hard requirements, or actions that are difficult to reverse. Reserve this for things that can actually break the build.

③ Changing the accent color

// IN styles.css

Find the :root block at the very top of styles.css. Change these three variables — everything else updates automatically:

:root {
  /* Change these three to retheme the entire accent: */
  --color-orange:    #f06a00;   /* ← your new accent */
  --color-orange-lt: #ff8c2a;   /* ← ~20% lighter    */
  --color-orange-dk: #c45500;   /* ← ~20% darker     */

  /* Leave the green palette as-is unless you want a full rebrand */
}

④ Adding a photo to a device section in devices.html

// TWO METHODS

Method A — External file (recommended for new pages): Place the image in the img/ folder next to your HTML files. Use a normal <img src="img/yourphoto.jpg"/> tag.

Method B — Base64 embedded (used in current pages for portability): Convert the image to base64 with a tool like base64.guru, then use src="data:image/jpeg;base64,...". This makes the HTML self-contained but larger.
