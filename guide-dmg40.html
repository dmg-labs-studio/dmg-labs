// OVERVIEW

What you're building

The DMG-40 is a compact real-time departure board for the trains and metros you actually take. Four 8×8 LED matrix panels sit behind a smoked acrylic diffuser inside a Braun-inspired FDM enclosure, showing the next two departures for any combination of stops you configure. The Pico W polls the transit API every 30 seconds over Wi-Fi and syncs its clock with NTP so the time is always accurate — even after a power cut.

The enclosure is a single-piece FDM print with a friction-fit back panel, so it requires no screws and opens easily for cable access. The red-on-black color scheme is a nod to the Nuremberg city colors that inspired the original.

This guide covers everything from printing to a working display. The firmware step assumes no prior MicroPython experience — all the scripts you need are in the download package.

The rendered DMG-40 in an urban European setting.

Also looks great in any industrial or Bauhaus-inspired ambience.

// TIP — sourcing the matrix modules
Buying a 4-in-1 pre-chained MAX7219 module (common on Amazon and AliExpress) saves you the daisy-chaining step and makes the wiring cleaner. The firmware is written for that configuration. If you buy individual modules, you'll need to solder the CLK/CS/DIN chain yourself.
// BUILD STEPS

Step by step

Work through these in order. Steps 1–3 are hardware, steps 4–6 are firmware, and steps 7–10 bring it all together. Allow the filament to cure before stressing the enclosure clips.

01
Print the enclosure

Open dmg40_enclosure_v2.stl in your slicer. The model is pre-oriented — print it flat-side-down with no supports. Recommended settings:

Layer height:   0.2 mm
Infill:         20 % (gyroid)
Walls:          3 perimeters
Bed temp:       60 °C  (PLA)
Nozzle temp:    215 °C (PLA)

Print time is around 2.5 hours on a Prusa MK4 at 0.2 mm. The front aperture has a 0.1 mm tolerance for the acrylic sheet — if your printer runs tight, scale X/Y by 1.002.

02
Assemble the LED matrix

If you bought individual 8×8 modules, daisy-chain them by connecting DOUT of each module to DIN of the next. If you have the 4-in-1 pre-chained board, skip this — it ships ready to wire.

Slide the acrylic diffuser into the front aperture from the inside. It should sit flush with no adhesive needed; the fit holds it in place. If it's loose, a thin strip of masking tape on the edges does the job neatly.

// NOTE — orientation
The matrix modules have a correct orientation for the text direction. The DIN connector faces right when the display is oriented horizontally. If your departure text scrolls backwards, rotate the module 180° and update ROTATE = True in config.json.
03
Wire up the Pico W

Connect the MAX7219 chain to the Pico W using the wiring diagram in wiring_diagram.pdf. The key connections are:

Matrix DIN  →  Pico GP11  (SPI0 MOSI)
Matrix CLK  →  Pico GP10  (SPI0 SCK)
Matrix CS   →  Pico GP9   (chip select)
Matrix VCC  →  Pico VBUS  (5 V)
Matrix GND  →  Pico GND

Use female–female Dupont wires to avoid any soldering. Route the wires along the inside wall of the enclosure before closing — there are two cable-guide clips printed inside the shell.

04
Flash the firmware

Download the latest stable version of the MicroPython distribution for the Pico 2W from here: micropython.org ↗ Hold the BOOTSEL button on the Pico W while connecting it via USB to your laptop. It mounts as a mass storage device. Drag the downloaded .uf2 micropython firmware file onto the drive — the Pico restarts and the drive disappears. That's it; MicroPython is now flashed onto the Pico.

Next, use Thonny IDE (free, cross-platform) to open and upload dmg40_firmware.py and config.json to the Pico's root filesystem. In Thonny: Run → Select Interpreter → MicroPython (Raspberry Pi Pico).

// FIRST TIME WITH THONNY?
Install it from thonny.org. Once connected to the Pico, you'll see the device's filesystem in the bottom panel. Right-click any file on your computer and choose "Upload to /" to copy it across.
05
Configure your transit API

Open config.json in any text editor. The key fields to fill in:

{
  "wifi_ssid":     "YOUR_NETWORK",
  "wifi_password": "YOUR_PASSWORD",
  "api_provider":  "VGN",
  "stop_ids":      ["510", "906"],
  "lines_filter":  ["U2", "S3"],
  "api_key":       ""
}

The firmware supports VGN (Nuremberg), VGA, DB, SNCF, TfL (London), and NYC MTA out of the box. The api_provider field selects the correct endpoint and data parser. Most European open transit APIs require no key — set api_key to an empty string if yours doesn't.

To find your stop IDs: search your stop on openstreetmap.org, click the stop, and note the ref tag. Alternatively, the stop_finder.py script in the download package queries the API by name and prints matching IDs.

06
Set the NTP clock

The clock syncs automatically on first boot as long as Wi-Fi is configured. Set your timezone offset in config.json:

"timezone_offset": 1    // CET — use 2 for CEST (summer)

The Pico W has no onboard RTC battery, so it re-syncs with NTP each time it powers on. If the display shows --:-- on boot, it means Wi-Fi hasn't connected yet — check your credentials and that the SSID is 2.4 GHz (the Pico W doesn't support 5 GHz).

07
Final wiring and cable routing

Route the USB-C cable through the back-panel slot before closing the enclosure. The slot is sized for a right-angle USB-C connector, which makes for a much cleaner exit — it's listed as optional in the BOM, but strongly recommended.

Tuck the Dupont wires into the two cable-guide clips on the inside of the enclosure. Give each wire a gentle tug to confirm nothing is loose before sealing up.

08
Close the enclosure

Align the back panel's four corner tabs with the slots in the main body. Press firmly until you hear a click from each corner — the friction-fit doesn't need force, just even pressure. If one corner won't seat, check for a wire caught in the gap.

Thread the four M2 × 6 screws into the back panel's recessed holes. They seat into self-tapping channels in the PLA — use a light hand on the final turns to avoid stripping.

09
First power-on

Plug into any USB-C 5 V power source. The display should show a brief boot animation, then the --:-- clock placeholder while it connects to Wi-Fi. Within 10–15 seconds you should see the real time followed by your first departure entry.

If nothing lights up: confirm the matrix VCC is on the Pico's VBUS pin (not 3V3) — the modules need 5 V. If the clock shows but no departures appear, check your stop_ids and API provider in config.json.

// IMPORTANT — power supply
The four LED matrix modules can draw up to 400 mA combined at full brightness. Use a proper phone charger or USB hub with a dedicated power supply — powering via a laptop USB port may cause flickering at high brightness.

10
Customising the display

The display cycles between two modes: a scrolling departure ticker and a real-time clock. You can adjust the cycle timing and brightness in config.json:

"brightness":      6,       // 0–15 (8 is a good indoor level)
"ticker_speed":    40,      // ms per scroll step — lower = faster
"clock_dwell":     8000,    // ms to show the clock between tickers
"display_mode":    "both"   // "ticker", "clock", or "both"

You can also add a second Wi-Fi network (useful if you travel with the device) by adding a wifi_fallback key with an SSID and password. The firmware tries the primary first and falls back automatically.

All changes take effect on the next reboot — power cycle or use Thonny's reset button. No reflashing needed.

// TROUBLESHOOTING

Common issues

troubleshoot.txt
Symptom Likely cause & fix
Display stays dark after power-on Check VCC is on VBUS (5 V), not 3V3. Re-seat the DIN/CLK/CS wires.
Text scrolls in the wrong direction Set "rotate": true in config.json and reboot.
Clock shows but no departures Wrong stop_ids or api_provider. Run stop_finder.py to confirm valid IDs.
Display flickers at high brightness Insufficient power supply. Use a 5 V / 1 A+ charger, not a laptop port.
Wi-Fi won't connect Pico W is 2.4 GHz only. Confirm the SSID is not a 5 GHz-only network. Check password for special characters.
Enclosure tabs won't seat Check for a wire caught in the seam. Scale the back panel STL to 99.8% if your printer runs large.
